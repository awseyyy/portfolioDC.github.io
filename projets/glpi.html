Situation professionnelle · Stage 1

Helpdesk et inventaire
centralisés avec GLPI 10.

Mise en production d'une instance GLPI 10 avec FusionInventory pour gérer à la fois le parc informatique (50+ équipements) et les tickets d'assistance niveaux 1 et 2 d'une PME.

Durée2 semaines (stage)
ContexteStage 1
EnvironnementDebian 12 · GLPI 10
BlocBloc 1

1. Contexte

Chez TechnoServices (stage 1), le support IT fonctionnait avec des e-mails et un vieux tableur Excel pour l'inventaire. Résultat : doublons, tickets oubliés, aucun suivi d'historique matériel. J'ai proposé la mise en place de GLPI 10, la solution open source leader en France pour l'ITSM.

2. Architecture

3. Installation

# Stack LAMP $ sudo apt install apache2 mariadb-server php php-mysql php-gd \ php-curl php-intl php-ldap php-xml php-zip php-bz2 php-mbstring -y # Base de données GLPI $ sudo mysql mysql> CREATE DATABASE glpi; mysql> CREATE USER 'glpi'@'localhost' IDENTIFIED BY 'MotDePasseFort'; mysql> GRANT ALL ON glpi.* TO 'glpi'@'localhost'; # Téléchargement et déploiement $ cd /tmp && wget https://github.com/glpi-project/glpi/releases/download/10.0.10/glpi-10.0.10.tgz $ sudo tar -xzf glpi-10.0.10.tgz -C /var/www/ $ sudo chown -R www-data:www-data /var/www/glpi

Puis passage dans l'assistant web pour finaliser la connexion à la base. Important : à la fin du wizard, on supprime /var/www/glpi/install/install.php et on active HTTPS (certificat Let's Encrypt).

4. Configuration

4.1 — Entités

Une entité racine "TechnoServices" avec deux sous-entités : Paris et Lyon. Chaque technicien voit uniquement son site.

4.2 — Profils & droits

ProfilDroitsPopulation

Super-AdminTout2 comptes, stockés en coffre

4.3 — Synchro LDAP avec l'Active Directory

Plutôt que de recréer les comptes, j'ai configuré la connexion LDAP vers DC01.techno.local (port 389, binding par compte de service svc_glpi). Résultat : tous les utilisateurs du domaine peuvent se connecter à GLPI avec leurs identifiants Windows, aucun mot de passe à gérer.

5. FusionInventory (inventaire automatique)

5.1 — Côté serveur

Installation du plugin FusionInventory for GLPI (fichier .tar.bz2 à copier dans /var/www/glpi/plugins), puis activation dans l'UI.

5.2 — Côté client Windows

Installation silencieuse via GPO (cf. projet AD) de l'agent :

msiexec /i fusioninventory-agent_windows-x64_2.6.msi /quiet ` SERVER=https://glpi.techno.local/plugins/fusioninventory ` RUNNOW=1 ADD_FIREWALL_EXCEPTION=1

Au prochain reboot, chaque poste apparaît automatiquement dans GLPI avec son matériel, ses logiciels installés, son numéro de série… un gain de temps spectaculaire par rapport à un inventaire Excel.

6. SLA & règles métier

Pour les tickets, j'ai créé deux SLA :

  • SLA-Critique : prise en charge sous 1h, résolution sous 4h. (Déclenchée par mot-clé « urgent » ou sur catégorie "Serveur").
  • SLA-Standard : prise en charge sous 8h, résolution sous 5 jours.

Des règles métier affectent automatiquement les tickets à la bonne file selon leur catégorie (Imprimante → équipe A, Réseau → équipe B, etc.).

7. Tableau de bord

Dans l'interface : vue globale des tickets en cours, temps moyen de résolution, taux de respect du SLA. J'ai exporté un rapport mensuel au format PDF pour le responsable.
