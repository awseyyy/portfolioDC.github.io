Segmentation d'un réseau
en 4 VLAN sur Cisco IOS.

Conception et mise en œuvre sur Cisco Packet Tracer d'un réseau d'entreprise segmenté en 4 VLAN (Administration, Production, Invités, Voix), avec trunking 802.1Q, routage inter-VLAN et listes de contrôle d'accès (ACL).

Durée1 semaine
ContexteAtelier pro
EnvironnementPacket Tracer 8.2
BlocBloc 2

1. Contexte & besoin

Une PME fictive de 60 postes connaît des problèmes de performance (broadcasts massifs) et de sécurité (tout le monde voit tout le monde). On me demande de segmenter le réseau pour :

  • Séparer les flux métier (admin, production) du réseau invité (Wi-Fi clients).
  • Isoler la téléphonie IP dans un VLAN dédié (QoS + sécurité).
  • Continuer à permettre le routage entre VLAN uniquement quand c'est nécessaire.

4. Configuration Cisco (extraits)

4.1 — Création des VLAN sur SW1

SW1> enable SW1# configure terminal SW1(config)# vlan 10 SW1(config-vlan)# name ADMIN SW1(config-vlan)# exit SW1(config)# vlan 20 SW1(config-vlan)# name PROD SW1(config-vlan)# exit SW1(config)# vlan 30 SW1(config-vlan)# name GUEST SW1(config)# vlan 40 SW1(config-vlan)# name VOICE

4.2 — Port d'accès avec téléphonie

! Exemple : PC branché sur un téléphone, téléphone branché sur le switch SW1(config)# interface FastEthernet0/5 SW1(config-if)# switchport mode access SW1(config-if)# switchport access vlan 20 SW1(config-if)# switchport voice vlan 40 SW1(config-if)# spanning-tree portfast

4.3 — Trunk entre switch et routeur

SW1(config)# interface GigabitEthernet0/1 SW1(config-if)# switchport mode trunk SW1(config-if)# switchport trunk encapsulation dot1q SW1(config-if)# switchport trunk allowed vlan 10,20,30,40

4.4 — Router-on-a-Stick sur R1

R1(config)# interface GigabitEthernet0/0 R1(config-if)# no shutdown R1(config-if)# interface GigabitEthernet0/0.10 R1(config-subif)# encapsulation dot1Q 10 R1(config-subif)# ip address 192.168.10.1 255.255.255.0 R1(config-subif)# interface GigabitEthernet0/0.20 R1(config-subif)# encapsulation dot1Q 20 R1(config-subif)# ip address 192.168.20.1 255.255.255.0 ! idem pour VLAN 30 et 40

5. ACL (isolation du VLAN invité)

Les invités doivent accéder à Internet, mais pas au réseau interne. Je crée une ACL étendue sur la sous-interface VLAN 30 :

R1(config)# ip access-list extended GUEST-ACL R1(config-ext-nacl)# deny ip 192.168.30.0 0.0.0.255 192.168.10.0 0.0.0.255 R1(config-ext-nacl)# deny ip 192.168.30.0 0.0.0.255 192.168.20.0 0.0.0.255 R1(config-ext-nacl)# deny ip 192.168.30.0 0.0.0.255 192.168.40.0 0.0.0.255 R1(config-ext-nacl)# permit ip any any R1(config-ext-nacl)# exit R1(config)# interface G0/0.30 R1(config-subif)# ip access-group GUEST-ACL in

6. Tests

  • ping PC-admin → PC-prod : OK (routage inter-VLAN fonctionne).
  • ping PC-guest → PC-admin : bloqué par l'ACL ✔
  • ping PC-guest → 8.8.8.8 (Internet) : OK ✔
  • show interface trunk → le trunk porte bien les VLAN 10, 20, 30, 40.
Apprentissage clé — le router-on-a-stick est la manière la plus économique d'apprendre le routage inter-VLAN. En production, un switch de niveau 3 serait préféré pour éviter le goulet d'étranglement sur un seul lien.
