Situation professionnelle · Stage 2

Mise en production d'un serveur
de supervision Zabbix.

Déploiement d'un serveur Zabbix 6 LTS sur Debian 12 pour superviser 40+ équipements (serveurs Windows/Linux, switchs SNMP, onduleurs) avec alertes e-mail et tableaux de bord temps réel.

Durée1 semaine (stage)
ContexteStage 2
EnvironnementDebian 12 · Zabbix 6 LTS
BlocBloc 1 · Bloc 2

1. Contexte & besoin

Pendant mon stage chez InfraCloud Solutions, j'ai participé à la mise en place d'une supervision centralisée. Jusqu'alors, les incidents étaient remontés par les utilisateurs : un serveur plantait, personne ne le voyait avant 9h le lendemain. L'objectif :

  • Surveiller en temps réel l'état des serveurs, switchs et onduleurs critiques.
  • Déclencher une alerte e-mail dès qu'une métrique dépasse un seuil.
  • Constituer un historique de performance pour justifier les prochains investissements matériels.

3. Installation du serveur Zabbix

Sur une VM Debian 12 fraîchement installée :

# 1. Ajout du dépôt officiel Zabbix 6 LTS $ wget https://repo.zabbix.com/zabbix/6.0/debian/pool/main/z/zabbix-release/zabbix-release_6.0-5%2Bdebian12_all.deb $ sudo dpkg -i zabbix-release_6.0-5+debian12_all.deb $ sudo apt update # 2. Installation du serveur, frontend, agent, pilote MySQL $ sudo apt install zabbix-server-mysql zabbix-frontend-php \ zabbix-apache-conf zabbix-sql-scripts zabbix-agent -y # 3. Création de la base $ sudo mysql -uroot -p mysql> create database zabbix character set utf8mb4 collate utf8mb4_bin; mysql> create user 'zabbix'@'localhost' identified by 'MotDePasseFort!'; mysql> grant all privileges on zabbix.* to 'zabbix'@'localhost'; mysql> set global log_bin_trust_function_creators = 1; # 4. Import du schéma $ zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql -uzabbix -p zabbix # 5. Mot de passe BDD dans /etc/zabbix/zabbix_server.conf, puis : $ sudo systemctl restart zabbix-server zabbix-agent apache2 $ sudo systemctl enable zabbix-server zabbix-agent apache2

L'interface est accessible sur http://<ip-serveur>/zabbix. Compte par défaut : Admin / zabbix (à changer immédiatement).

4. Ajout des hôtes supervisés

4.1 — Agent Zabbix sur un serveur Linux

$ sudo apt install zabbix-agent -y $ sudo nano /etc/zabbix/zabbix_agentd.conf Server=192.168.50.10 (IP du serveur Zabbix) ServerActive=192.168.50.10 Hostname=WEB01 $ sudo systemctl restart zabbix-agent

Puis dans l'UI : Configuration → Hosts → Create host → lier le template Linux by Zabbix agent.

4.2 — Supervision SNMP d'un switch Cisco

Sur le switch : snmp-server community techno-ro RO.
Dans Zabbix, j'ajoute l'hôte avec l'interface SNMP, la bonne communauté, et je lie le template Cisco IOS SNMP. Les métriques utiles : trafic par port, statut UP/DOWN, température, charge CPU.

6. Tableau de bord

J'ai créé un dashboard "NOC TechnoServices" avec :

  • Map réseau cliquable (hôtes colorés selon sévérité).
  • Top 10 des hôtes en charge CPU.
  • Graphe de trafic des liens WAN.
  • Tableau des 24 dernières alertes.
Astuce stage — j'ai affiché ce dashboard sur un écran mural du bureau technique. Les collègues m'ont dit que ça avait changé la dynamique de l'équipe : on est passés de « réactif sur réclamation » à « proactif sur incident ».

8. Bilan

Ce projet m'a appris qu'on ne supervise pas ce qu'on ne comprend pas : avant de créer un déclencheur, il faut savoir quelle métrique mesurer et quel seuil est réaliste. Zabbix est puissant mais vite noyé sous les alertes si on en met trop : j'ai dû faire un vrai travail de tri pour garder uniquement les alertes actionnables.
